\# Use Case Overview



\## 1. System Name



Horse Racing Management System



\## 2. Primary Actors



The main actors of the system include:



\* Admin

\* Horse Owner

\* Jockey

\* Race Referee

\* Spectator



\## 3. Use Case Summary



This document describes the main use cases of the Horse Racing Management System. The system supports horse racing tournament management, horse registration, jockey assignment, race scheduling, result tracking, ranking, prediction, prize management, and referee reporting.



\## 4. Use Cases by Actor



\### 4.1 Admin



Admin is responsible for managing the whole system.



Main use cases:



\* Manage user accounts

\* Manage user roles and permissions

\* Manage tournaments

\* Manage races and race schedules

\* Approve horse registrations

\* Manage horses and jockeys

\* Assign referees to races

\* Publish race results

\* Manage prediction activities



\### 4.2 Horse Owner



Horse Owner is responsible for managing horses and registering them for tournaments or races.



Main use cases:



\* Register account

\* Manage horse information

\* Register horse for tournament

\* Select or invite jockey for horse

\* Confirm jockey participation

\* View race schedule of registered horses

\* View race information

\* View race results, rankings, and horse prizes



\### 4.3 Jockey



Jockey is responsible for controlling horses in races.



Main use cases:



\* Register jockey account

\* Receive invitation from horse owner

\* Accept or reject race invitation

\* Confirm race participation

\* View assigned races

\* View horse information

\* View personal race schedule

\* View personal race results, rankings, and achievements



\### 4.4 Race Referee



Race Referee is responsible for checking and confirming race activities.



Main use cases:



\* Check horse information before race

\* Monitor race process

\* Record race violations

\* Handle race violations

\* Confirm race result

\* Create referee report



\### 4.5 Spectator



Spectator is responsible for viewing race information and predicting race results.



Main use cases:



\* View tournament information

\* View race schedule

\* Follow race results

\* View rankings

\* Predict race result

\* View prediction result

\* Receive prediction reward notification



\## 5. Main Use Case Groups



The system use cases can be grouped into the following modules:



\* Authentication and Authorization

\* User and Role Management

\* Horse Management

\* Jockey Management

\* Tournament Management

\* Race Management

\* Registration Management

\* Race Result and Ranking Management

\* Referee Report Management

\* Prediction Management

\* Prize and Reward Management

\* Notification Management
